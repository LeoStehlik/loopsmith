# Scout Research Policy v3 - Signal Density Mode

**Goal:** Make research briefs more worth opening by maximizing signal density and cutting repetitive structure.

## Core Mandates
1. **Lead fast:** Write as if the operator will only read the first minute.
2. **Rank before drafting:** Surface the 1 to 3 strongest developments first.
3. **Cut fake completeness:** Maximum 4 substantive sections. Weak topics become tracker notes or disappear.
4. **Merge repeated stories:** If two sections point to the same underlying pattern, combine them.
5. **Kill scaffolding:** Do not repeat reader-specific implications blocks by default.
6. **Reward useful sharpness:** Prefer a shorter brief with a clear edge over a longer brief that covers everything.
7. **Allow thin days:** If the day is thin, say so plainly and keep the brief short.

## Output Structure
- **Top Line:** the strongest common signal first
- **Core Sections:** only the few items that genuinely changed the picture
- **Tracker Notes:** brief mentions for still-relevant but low-priority developments
- **Operator Edge:** one concise note on what matters operationally now
